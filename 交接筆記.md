## 目前主要功能

- 永豐存股放在首頁最上方。
- 永豐存股可同步 API 庫存，顯示股數、買入金額、目前金額、盈虧金額、目前股價、漲跌。
- 開盤期間可每 30 或 60 秒同步一次。
- 每次同步後會重算停損、停利、移動停利。
- 移動停利只往上，不往下放。
- 停損或移動停利觸發時，用彈窗提醒，需手動關閉。
- 本金還虧損時，不提醒賣出。

## 每日流程規則

目前設定如下：

```text
08:30 更新資料
09:05 只排除風險
09:15 初步確認
09:30 才允許買進
盤中每 30/60 秒管理持股
13:30 收盤後停止同步
```

後端已改成 08:30 後才會自動跑每日更新，而且一天只跑一次。前提是 `python server.py` 有開著，且 FinMind token 已設定。

## 妖股雷達規則

妖股雷達目前會顯示四個摘要：

- 掃描檔數
- 正在轉強檔數
- 可觀察檔數
- 真正可買檔數

真正可買檔數只有在 09:30 後，且開盤二次確認通過，才會大於 0。

目前買進邏輯：

- 收盤後產生候選名單。
- 隔天開盤先排除跳空過高、開高走低、流動性不足、過熱。
- 09:30 後才允許顯示「可買 1 張」。
- 不自動下單，只做輔助判斷。

## 題材熱度雷達

題材熱度雷達會依題材分組，顯示：

- 題材熱度
- 可觀察檔數
- 平均妖股分數
- 5 日 / 20 日動能
- 量能
- 代表股票
- 新手判斷
- 新聞支撐按鈕

目前 OpenAI 可用於白話分析，Perplexity 可用於新聞摘要。

## 個股完整分析

個股完整分析目前有兩種模式：

- 新手簡易版：用圖示化、白話方式顯示。
- 完整版：顯示所有模型、技術、籌碼、大盤、風險資料。

每檔股票會顯示：

- 技術分數
- 籌碼分數
- 大盤分數
- 模型分數
- 風險分數
- 總分
- 買進條件是否通過
- 賣出條件是否觸發

停損、停利、移動保護價也會畫在 K 線圖上。

## 模型狀態

目前後端正式模型包含：

- Logistic Regression
- XGBoost
- LightGBM
- Gradient Boosting（LightGBM 類型）
- Isolation Forest 異常偵測
- Learning to Rank 排名模型

目前 `model.pkl` 已保存訓練結果。

模型用途：

- Logistic Regression：基準模型。
- XGBoost / LightGBM / Gradient Boosting：非線性分類，判斷未來上漲機率。
- Isolation Forest：偵測異常強勢或異常風險。
- Learning to Rank：做候選股票排序。

## 目前風控邏輯

欄位名稱已改成較新手化：

- 風控停損價
- 目標停利價
- 移動保護價
- 模型調整原因

停損停利會參考：

- ATR 波動
- 模型看好程度
- 大盤多空
- 個股是否強於大盤
- 是否已獲利
- 是否跌破移動保護價
